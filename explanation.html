Explanation of Metrics (69)

Here are the different metrics collected through SonarQube's API for a given Java project:

Metric Definition
1. sqale_index The SQALE index that measures the overall code quality based on various criteria such as maintenance, reliability, security, and efficiency.
2. code_smells The number of "code smells," indicating pieces of code that may require improvements in terms of readability, maintainability, or performance.
3. sqale_debt_ratio The technical debt ratio based on the SQALE model. It indicates how much technical work is needed to achieve an ideal code state.
4. bugs The number of bugs identified in the code, representing functional or logical issues.
5. vulnerabilities The number of vulnerabilities identified in the code, indicating security weaknesses that could be exploited by attackers.
6. duplicated_lines_density The density of duplicated lines of code, an indicator of code quality in terms of reuse and duplication.
7. coverage The code coverage rate by tests, expressed as a percentage. It measures the proportion of code covered by tests compared to the total code.
8. complexity The overall complexity of the code, measured based on code structure and dependencies.
9. cognitive_complexity The cognitive complexity of the code, evaluating complexity based on human understandability rather than purely technical metrics.
10. ncloc (Non Commented Lines Of Code) The number of lines of code without comments, allowing evaluation of the actual code size.
11. comment_lines The number of lines containing comments, providing insight into documentation and explanations in the code.
12. comment_lines_density The density of comment lines compared to the code, expressed as a percentage.
13. duplicated_blocks The number of duplicated code blocks, showing sections of code that repeat in the application.
14. duplicated_files The number of identical or very similar files in the code.
15. duplicated_lines The number of lines of code that are identical or very similar, indicating a duplication issue.
16. new_bugs The number of new bugs identified since the last analysis.
17. new_code_smells The number of new "code smells" identified since the last analysis.
18. new_coverage The new code coverage rate by tests since the last analysis.
19. new_duplicated_blocks The number of new duplicated code blocks since the last analysis.
20. new_duplicated_lines The number of new duplicated lines of code since the last analysis.
21. new_duplicated_lines_density The density of new duplicated lines of code since the last analysis.
22. new_maintainability_rating The new maintainability rating since the last analysis.
23. new_reliability_rating The new reliability rating since the last analysis.
24. new_security_hotspots The number of new security hotspots identified since the last analysis.
25. new_security_rating The new security rating since the last analysis.
26. new_technical_debt The new technical debt since the last analysis.
27. new_vulnerabilities The number of new vulnerabilities identified since the last analysis.
28. alert_status The overall alert status of the project.
29. quality_gate_details Details on the code quality according to the quality gate.
30. reliability_rating The code reliability rating.
31. security_rating The code security rating.
32. security_hotspots_reviewed_status The security hotspots review status.
33. sqale_rating The code quality rating based on the SQALE model.
34. effort_to_reach_maintainability_rating_a The estimated effort to reach the maximum maintainability rating.
35. violations The total number of code non-conformities to quality rules.
36. blocker_violations The number of blocking non-conformities.
37. critical_violations The number of critical non-conformities.
38. major_violations The number of major non-conformities.
39. minor_violations The number of minor non-conformities.
40. info_violations The number of information non-conformities.
41. new_violations The number of new non-conformities since the last analysis.
42. new_blocker_violations The number of new blocking non-conformities since the last analysis.
43. new_critical_violations The number of new critical non-conformities since the last analysis.
44. new_major_violations The number of new major non-conformities since the last analysis.
45. new_minor_violations The number of new minor non-conformities since the last analysis.
46. new_info_violations The number of new information non-conformities since the last analysis.
47. false_positive_issues The number of issues wrongly identified as problems.
48. open_issues The total number of open issues.
49. confirmed_issues The number of confirmed issues.
50. reopened_issues The number of reopened issues.
51. wont_fix_issues The number of issues marked as not needing correction.
52. classes The total number of classes.
53. files The total number of files.
54. functions The total number of functions.
55. lines The total number of lines of code.
56. ncloc_language_distribution The code distribution by language.
57. projects The total number of projects.
58. statements The total number of statements.
59. public_api The total number of public API.
60. public_undocumented_api The total number of undocumented public API.
61. comment_lines_density The density of comment lines compared to the code.
62. public_documented_api_density The density of documented public API.
63. lines_to_cover The number of lines to cover by tests.
64. uncovered_lines The number of lines of code not covered by tests.
65. line_coverage The code line coverage rate by tests, expressed as a percentage.
66. conditions_to_cover The number of conditions to cover by tests.
67. uncovered_conditions The number of conditions not covered by tests.
68. branch_coverage The branch coverage rate by tests, expressed as a percentage.
69. new_lines The number of new lines of code added since the last analysis.

Some Java projects may not have all these metrics, which is why there are fewer in some project metric datasets.
