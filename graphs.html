Knowing the AVG and STDEV values and assuming a normal distribution for the collected data (i.e., that most values are concentrated in the middle rather than the margins of the data set), we also know the two margins of the typical values interval for a metric and the threshold for very high values. These are:
• Lower margin: AVG - STDEV
• Higher margin: AVG + STDEV
• Very high: (AVG+STDEV )*1.5, i.e, we consider a value to be very high if it is 50% higher than the threshold for a high value.



CYCLO Graph

CYCLO: The CYCLO metric, or cyclomatic complexity, or McCabe measure is used to measure the complexity of a project. It reflects the number of decisions in an algorithm by counting the number of linearly independent paths through a program. The usefulness of cyclomatic complexity lies in its ability to provide an estimate of the complexity of a software product. It can be used to assess the structural complexity of a program, which can help identify areas of code that may require redesign or revision. Moreover, a high cyclomatic complexity value may indicate a lack of modularity, which can make the code more difficult to maintain and understand. Finally, cyclomatic complexity can be used to estimate the effort required to test a program, as a higher number of paths may require a larger number of unit tests.

NOM Graph

NOM: The NOM metric is the total number of methods or functions in a project. The usefulness of the NOM metric lies in its ability to provide an estimate of the complexity of a software product. It can be used to assess the structural complexity of a program, which can help identify areas of code that may require redesign or revision. Moreover, a high NOM value may indicate a lack of modularity, which can make the code more difficult to maintain and understand. Finally, the NOM metric can be used to estimate the effort required to test a program, as a higher number of methods may require a larger number of unit tests.

NCLOC Graph

NCLOC: The NCLOC metric is the number of lines excluding comments and blank lines. It's an average of the number of lines of code per project. The usefulness of the NCLOC metric lies in its ability to provide an estimate of the physical size of a software product. It can be used to assess the space needed for storage, the number of pages needed to print the program, and the management of on-screen viewing. In addition, the internal attributes of the software product, such as size measured by NCLOC, can impact its external attributes. A good internal structure can lead to better product quality.

AWM Graph

AWM (Average Weighted Method): The AWM metric measures the average complexity of a class’s methods. A high value can indicate a “code smell” called “Complex Method”, where the methods are too complex. This metric can help identify parts of the code that could benefit from a redesign to simplify logic and improve readability. Moreover, by reducing the complexity of methods, one can often improve other attributes of the software, such as its ease of testing and maintenance. Ultimately, the goal is to create code that is not only functional but also clean and efficient.
